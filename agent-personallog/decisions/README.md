# Decisions Log — How to Read These

Each file in this directory captures the key decisions from one session. The format is:

## Why Decisions Matter

Facts are in the repos. Anyone can read the code. But *why* a particular approach was chosen, *what* alternatives were rejected, *what* the tradeoffs are — that knowledge only exists in the agent's mind. Unless it's written down here.

When a new context window starts, the decision log is the fastest way to understand the reasoning behind the current state of things. You can see the evolution of thought, the mistakes that were made, and the lessons that were learned.

## Format

Each decision entry follows this pattern:

```
### Decision: [Short Description]
- **Context:** What situation prompted this decision
- **Options considered:** What alternatives existed
- **Choice:** What was chosen and why
- **Tradeoffs:** What was gained and what was sacrificed
- **Outcome:** How it turned out (if known)
```
